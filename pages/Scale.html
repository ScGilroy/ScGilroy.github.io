Learning about strain gauges, I wanted to utilize this technology in a project. A digital scale that utilizes multiple strain gauges to measure beam deflection does this, and allows for weight and thrust to be measured.

Skills Utilized:

  • Inventor
  • Circuitry
  • Material Properties

Overview

The goal of this project was to create a scale utilizing an Arduino to measure and record different thrust and weight values applied. By knowing the material the force is applied upon, strain gauges can be utilized to measure the deflection in the beam. Strain gauges allow for the interface between the analog and digital world by utilizing a flexible housing which varies the resistance as the material is deflected. The small changes in resistance can be measured by an Arduino and can be converted into applied force.

Arduino vs DAQ

Both devices have the capability of reading in the resistance measurement and converting the value to a force. There is a major difference between these devices for their measurement capabilities. The Arduino Uno's analog pins are limited to 10 bits resulting in a total of 1024 output bins. A DAQ often has a 32-bit system leading for over 4 billion output bins. This results in a much finer resolution for the DAQ but these devices are much more expensive than an Arduino Uno. To avoid this drastic decrease in precision between the devices, a Hx711 Load Cell Amplifier increases the precision to approximately 16.7 million bins. Although this is not nearly as precise as a DAQ, the weights associated with the light-weight scale should make the precision of the Hx711 good enough.

Theory and Schematic

Strain gauges are a marvel for interfacing deflections in beams to an applied force. To maximize the potential of a strain gauge, these variable resistors are often used in pairs of two or four. They are used in a Wheatstone Bridge configuration in pairs of two or four. With a four-strain gauge configuration, two gauges are used to measure deflections of each axis of the material. Overall the scale utilizes an Arduino Uno, strain gauges, and the Hx711. A 16x2 LCD screen with some buttons can be added to the circuit to allow the user to record and read forces. Using Fritzing, I created a schematic to do this with the various components previously mentioned.

Aluminum Extrusion vs Load cell

Instead of using a premade load cell, a thin piece of aluminum extrusion can be used. Four strain gauges can be applied with glue, two sets perpendicular to each other. By using the stress-strain relationship, different calibrated weights can be applied to calibrate the device. Afterward, resistance values measured from the strain gauges can be converted into force measurements via a constant multiplying factor.

I utilized this technique in a course to acquire force measurements from a Wheatstone Bridge configuration of strain gauges. The resistance values were measured into a 32-bit DAQ and a program written with LABVIEW displayed the force values realtime. For a personal project and having no way to machine metal, I decided that using a load cell was the easiest. Therefore, the design stands utilizing a load cell with pre-applied strain gauges.

Model

The housing itself was modeled in Autodesk Inventor which features a standard 5kg load cell. The design was made to be modular and hold the various components such as an LCD screen.

Conclusion

The design uses four posts with an m5 bolt runs through and screws into the bottom feet. The platform where the load is applied is also made of aluminum to prevent any warping wood has. Additionally, a notch is cut out on the bottom to allow a method of fastening a load to the platform without having it slide. The two platforms are made of wood which frames the device in as well as holds the electronics. The load cell itself can be swapped out to accommodate different measurement ranges. Although there are load cells that can measure hundreds of kilograms, this device can support light-weight measurements due to the wood as well as the Hx711's precision.
